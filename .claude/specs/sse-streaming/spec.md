# SSE 流式输出 spec.md

> 本文件回答"要做什么"。只描述需求、用户故事和验收标准，不讨论具体技术方案。

## 目标

为 KB Copilot 的问答能力增加 SSE 流式输出，用户在提问后可以实时看到回答逐 token 生成，而非等待完整响应后一次性展示。流式过程中需要支持 Markdown 和代码块的实时渲染，并允许用户中断生成或基于上一轮上下文重新生成回答。

本能力是 MVP3 基础设施增强的一部分，目标是将用户等待体验从"提交-等待-展示"升级为"提交-流式呈现-交互控制"。

## 用户故事

- 作为知识库使用者，我希望提问后能立即看到回答开始逐字输出（打字机效果），以便感知系统正在工作，减少等待焦虑。
- 作为知识库使用者，我希望流式输出过程中 Markdown 格式和代码块能实时渲染，以便在回答尚未完成时就已经可读。
- 作为知识库使用者，我希望可以中断正在进行的回答生成，以便在不需要当前回答时及时停止，节省 token 消耗。
- 作为知识库使用者，我希望对不满意的回答可以基于上一轮上下文重新生成，以便快速获得更好的答案。

## EARS 需求

- 当用户提交问题时，系统应通过 SSE 连接逐 token 推送回答内容。
- 当回答生成完成时，系统应发送包含引用来源的完成事件。
- 当生成过程中发生错误时，系统应通过 SSE 发送错误事件并关闭连接，前端展示错误信息。
- 当用户点击"中断"时，系统应中断 LLM 生成并关闭 SSE 连接，已生成的内容保留在聊天记录中。
- 当用户点击"重新生成"时，系统应基于上一轮用户问题重新发起流式生成，并用新回答替换之前的回答。
- 如果 SSE 连接意外断开，前端应展示连接中断提示。

## API 需求

```text
POST /api/v1/kbs/{kb_id}/chat/stream
  -> SSE 事件流:
     event: token       data: {"token": "..."}
     event: sources     data: {"sources": [...]}
     event: done        data: {"conversation_id": "...", "message_id": "..."}
     event: error       data: {"error": "...", "detail": "..."}

POST /api/v1/kbs/{kb_id}/chat/{conversation_id}/regenerate
  -> 复用已有问题，重新流式生成回答
```

现有 `POST /api/v1/kbs/{kb_id}/chat` 保持不变，作为非流式的降级路径。

## 数据需求

SSE 事件流：

- `token` 事件：`{"token": "<文本片段>"}` — 单个或少量 token 文本。
- `sources` 事件：`{"sources": [{"doc_id", "filename", "chunk_index", "score", "content"}]}` — 引用来源列表。
- `done` 事件：`{"conversation_id": "<id>", "message_id": "<id>"}` — 生成完成，携带已持久化的会话和消息 ID。
- `error` 事件：`{"error": "<错误码>", "detail": "<可读描述>"}` — 发生错误。

ChatStreamRequest：

- `question`：必填，用户问题。
- `top_k`：可选，检索数量，默认 5。
- `conversation_id`：可选，会话 ID。

RegenerateRequest：

- `top_k`：可选，检索数量，复用原 question。

## 验收标准

- 前端提问后，回答以打字机效果逐步出现，而非等待后一次性出现。
- Markdown 内容（标题、列表、加粗、链接等）在流式过程中实时渲染。
- 代码块在流式过程中实时渲染，保持语法高亮（MVP3 使用 react-markdown + 基础代码块样式即可，不要求语言级高亮）。
- 用户可以点击"停止生成"按钮中断当前回答。
- 中断后已输出的内容保留在聊天中。
- 用户可以对已完成的回答点击"重新生成"，获得新的流式回答。
- SSE 连接异常断开时，前端给出可感知的提示。
- 原有非流式接口功能不受影响。

## 不包含

- WebSocket 双向通信 — 使用单向 SSE 足够。
- 与非流式接口共用一个 endpoint 通过参数切换 — 使用独立 endpoint 保持逻辑清晰。
- 从前端 abort SSE 后自动回退非流式 — 中断即停止，不换路径重新请求。
- 语言级代码高亮（highlight.js / Prism）— MVP3 不做，后续可加。
- 断线重连自动续传 — MVP3 不做，断连后提示用户刷新即可。
