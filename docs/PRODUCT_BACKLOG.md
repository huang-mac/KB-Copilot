# Product Backlog

本文件用于收集零散但有价值的产品体验优化点。它不等同于当前迭代承诺；进入某个 MVP 或 spec 后，再同步到对应的 `docs/MVP*.md` 与 `.claude/specs/<feature>/tasks.md`。

## 使用规则

- 小体验、小交互、文案、布局优化先放这里。
- 已决定纳入当前 MVP 的事项，同步到对应 MVP tasks。
- 已完成的事项保留简短记录，避免后续重复讨论。
- 涉及接口、数据模型、权限、安全或架构边界的事项，不只放 backlog，必须补 spec。
