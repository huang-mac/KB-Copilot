# MVP4 企业化与生产治理

> **状态：规划中** 📌
>
> MVP4 建立在 MVP3 的基础设施能力之上，重点补齐企业内部试用和生产治理需要的安全、隔离、权限、配置和审计能力。

## 版本定位

MVP4 的目标是把 KB Copilot 从“基础设施完善的知识库问答底座”升级为“可进入企业内部小范围试用的受控系统”。

MVP4 不优先新增复杂检索能力，而是把 MVP3 已经建立的检索、路由、异步索引、MySQL 和监控能力纳入更完整的企业治理边界。

## 计划新增能力

- **API Key / JWT 鉴权**：先支持服务端 API Key，后续扩展登录态和 JWT。
- **多租户隔离**：按租户隔离知识库、文档、对话、索引任务和向量 payload。
- **权限过滤**：检索时根据用户、角色或文档权限过滤可访问内容。
- **管理员配置页**：管理模型配置、租户、知识库、权限策略和基础指标。
- **模型配置中心**：支持不同知识库或租户配置不同 LLM / Embedding / rerank 模型。
- **真实工具接入**：在 MVP3 mock 工具基础上接入真实 HTTP 工具；MCP 远程工具接入作为可选扩展。
- **审计日志**：记录文档操作、问答请求、工具调用、配置变更和权限命中情况。
- **生产部署增强**：补充 Nginx、HTTPS、健康检查、日志采集、备份恢复和容量建议。

## 后端改进

- 增加认证依赖，保护文档管理、问答、工具调用、指标和管理配置接口。
- 扩展元数据模型，增加 `tenant_id`、`user_id`、`role`、`permission_scope` 等字段。
- 扩展 Qdrant payload，支持租户、知识库、文档权限和可过滤字段。
- 增加权限过滤服务，统一应用于检索、文档列表、会话列表和工具调用。
- 增加模型配置 repository 和模型路由服务。
- 增加管理端 API，用于租户、知识库、模型、权限和指标管理。
- 增加审计日志 repository，支持按租户、用户、动作和时间范围查询。
- 扩展真实工具 adapter，支持 HTTP 工具调用；MCP 工具发现和调用作为后续可选能力。

## 前端改进

- 增加登录或 API Key 配置入口。
- 增加管理员配置页，用于管理租户、知识库、模型和权限策略。
- 文档管理页支持按租户、知识库、文档类型、上传时间和索引状态筛选。
- 问答页展示当前租户、当前知识库和可访问范围。
- 检索详情中展示权限过滤后的命中结果和被过滤数量。
- 增加审计日志查询页面，便于排查文档、问答和配置变更。

## 验收标准

- 未认证请求不能访问受保护的文档管理、问答、工具和管理接口。
- 不同租户之间的文档、对话、索引任务和检索结果互相隔离。
- 用户只能检索自己有权限访问的文档。
- 管理员可以配置模型、租户、知识库和基础权限策略。
- 审计日志能追踪文档上传、删除、重新索引、问答、工具调用和配置变更。
- 真实 HTTP 工具调用具备超时、错误处理和结构化返回。
- 生产部署文档可以指导企业内部小范围试用。
