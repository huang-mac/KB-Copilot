# MVP3 生产增强计划

## 版本定位

MVP3 面向生产增强，目标是在 MVP2 的轻量可用基础上，提升检索效果、索引稳定性、安全隔离和可观测性。

该版本适合从项目展示走向企业内部小范围试用。

## 计划新增能力

- **混合检索**：结合 BM25 关键词检索和 Qdrant 向量检索。
- **RRF 融合排序**：融合多路召回结果，提升专有名词、错误码、单据号等查询效果。
- **可选 rerank**：对 Top-K 候选片段进行二次排序。
- **异步索引**：使用任务队列处理大文档解析、切分和入库。
- **API Key / JWT 鉴权**：从简单 API Key 起步，后续可扩展到登录态和 JWT。
- **多租户隔离**：按租户隔离知识库、文档、对话和向量 payload。
- **权限过滤**：检索时根据用户角色或文档权限过滤结果。
- **模型路由**：支持不同知识库或租户配置不同 LLM / Embedding。
- **监控指标**：记录请求耗时、检索耗时、模型调用耗时、错误率和 token 用量。
- **部署增强**：补充生产环境 Docker Compose、Nginx、健康检查和日志配置。

## 后端改进

- 引入异步任务队列处理索引任务。
- 增加混合检索服务，统一封装向量召回、关键词召回、融合排序和 rerank。
- 扩展元数据模型，支持租户、权限和文档状态。
- 增加认证依赖，保护文档管理、问答、管理配置等接口。
- 增加模型配置表或配置中心能力。
- 增加结构化日志和 Prometheus 指标。

## 前端改进

- 展示索引任务进度。
- 支持按知识库、文档类型、上传时间筛选文档。
- 支持查看检索详情，例如命中片段、召回方式、rerank 分数。
- 增加管理员配置页，用于管理模型、租户和权限。

## API 计划

```text
GET    /api/v1/index-jobs/{job_id}
POST   /api/v1/kbs/{kb_id}/search
POST   /api/v1/auth/login
GET    /api/v1/auth/me
GET    /api/v1/admin/metrics
GET    /api/v1/admin/model-configs
POST   /api/v1/admin/model-configs
```

## 验收标准

- 错误码、业务编号、产品型号等精确查询比纯向量检索更稳定。
- 大文档上传不会阻塞 HTTP 请求。
- 文档管理和问答接口具备基础鉴权能力。
- 用户只能检索自己有权限访问的文档。
- 管理员可以看到基础运行指标。
- 生产部署文档可以指导小规模内部试用。

## 后续方向

MVP3 之后可以继续扩展：

- 企业 SSO / JWT 对接。
- 审计日志。
- 更细粒度的文档权限。
- 多模型成本统计。
- 与 ERP、WMS、客服系统的业务集成。
